Overview

This is the Constructor.io .NET SDK - a client library for interacting with Constructor.io's search-as-a-service platform. Constructor.io provides AI-powered search, autocomplete, recommendations, and browse functionality optimized for e-commerce.

Common Commands

Testing

# Run all tests
cd src/Constructorio_NET.Tests/
dotnet test
# Run a specific test file
dotnet test --filter "FullyQualifiedName~BrowseTests"
# Run a specific test method
dotnet test --filter "FullyQualifiedName~BrowseTests.GetBrowseResultsShouldReturnResultWithVariationsMapValueCount"

Code Coverage

# Initial setup (first time only)
dotnet husky install
# Run code coverage
dotnet husky run -g coverage

Building

# Build in Release mode (default)
./build.sh
# Build in Debug mode
./build.sh Debug
# Build with tests
./build.sh --test
# Build with clean
./build.sh --clean
# Combine options
./build.sh Release --test --clean

The build script creates a NuGet package in the ./artifacts directory.

Publishing

# Set your NuGet API key
export NUGET_API_KEY="your-api-key"
# Dry run to verify package details
./publish.sh --dry-run
# Publish to NuGet
./publish.sh

Architecture

Project Structure

  • src/constructor.io/ - Main SDK library (targets netstandard2.0)
    • client/ - Core ConstructorIO client and module implementations
      • ConstructorIO.cs - Main client entry point
      • modules/ - API module implementations (Autocomplete, Browse, Search, Recommendations, Catalog, Tasks, Items, Quizzes)
    • models/ - Request/response models organized by module (Autocomplete, Browse, Search, Recommendations, Catalog, Tasks, Items, Quizzes, common)
    • utils/ - Shared utilities (Constants, Helpers, JSON handling)
  • src/Constructorio_NET.Tests/ - Test project (targets net8.0)
    • client/modules/ - Test files mirroring the module structure (AutocompleteTests.cs, BrowseTests.cs, SearchTests.cs, RecommendationsTests.cs, CatalogTests.cs, TasksTests.cs, ItemsTest.cs, QuizzesTests.cs)
    • Uses NUnit 3.14.0 for testing

Core Modules

The SDK is organized into 8 main modules accessible through the ConstructorIO client:

  • Autocomplete - Get autocomplete suggestions and results
  • Browse - Browse results by filters, groups, collections, or item IDs
  • Search - Full-text search with filtering, faceting, and sorting
  • Recommendations - Get personalized product recommendations by pod ID
  • Catalog - Upload and manage product catalogs (items, variations, item_groups)
  • Tasks - Track and retrieve catalog ingestion task status
  • Items - Create, update, delete, and retrieve individual catalog items
  • Quizzes - Quiz-related functionality

Key Concepts

VariationsMap

Used across Autocomplete, Browse, Search, and Recommendations to control how product variations are aggregated and returned:

  • GroupBy rules: Define how variations are grouped (e.g., by URL)
  • Value rules: Define aggregation for variation attributes with 5 aggregation types:
    • first - First value encountered
    • min - Minimum value
    • max - Maximum value
    • all - All values as array
    • value_count - Count of distinct values matching a criteria
  • DType: Response format - object (default) or array
  • FilterBy: Filter variations using JSON expressions

HttpClient Management

The SDK supports two patterns:

  1. Static HttpClient (default): Uses a shared static instance for backwards compatibility
  2. IHttpClientFactory (recommended): Inject HttpClient via dependency injection for better resource management

Request/Response Pattern

All API operations follow a consistent pattern:

  1. Create a typed request object (e.g., SearchRequest, BrowseRequest)
  2. Configure request parameters (pagination, filters, sorting, etc.)
  3. Optionally add UserInfo for personalization
  4. Optionally add VariationsMap for variation control
  5. Call the module method to get a typed response

User Personalization

Most endpoints accept a UserInfo object containing:

  • ClientId (device ID) - Unique device identifier
  • SessionId - User session identifier This enables personalized and optimized results.
